PHP基础

Jan 25 2020

PHP( Hypertext Preprocessor, 超文本预处理器 )是一种通用开源脚本语言、创建动态交互性站点的强有力的服务器端脚本语言、弱类型语言。

通用开源：PHP 可免费下载使用。

创建动态交互性站点：PHP 可以生成动态页面内容、创建打开读取写入关闭服务器上的文件、收集表单数据、发送和接收 cookies、添加删除修改查找数据库中的数据等等功能。

服务器端：PHP 代码在服务器上执行

弱类型：PHP 会根据变量的值，自动把变量转换为正确的数据类型。而强类型的编程语言中，我们必须在使用变量前先声明(定义)变量的类型和名称。

脚本语言： 一种编程语言，用来控制软件应用程序，脚本通常以文本保存，只在被调用时进行解释或编译。

PHP 文件可包含文本、HTML、JavaScript代码和PHP 代码。PHP 代码在服务器上执行，结果以纯HTML 形式返回给浏览器。

PHP 语法

PHP脚本可以放在文档中的任何位置。PHP脚本以 < ? php 开始，以 ? > 结束。PHP中的每个代码行都必须以分号结束。分号是一种分隔符，用于把指令集区分开来。PHP文件通常包含 HTML 标签和一些 PHP脚本代码。PHP语句区分大小写。

PHP 变量

变量是用于存储信息的”容器”。

PHP 变量的引用

引用(别名)，不同的名字访问同一个变量内容， 指向同一个变量 。在变量、函数、对象等前加上 & 符号。

PHP 超级全局变量

超级全局变量在PHP 4.1.0之后被启用, 是PHP系统中自带的变量，在一个脚本的全部作用域中都可用。

PHP 常量

常量值被定义后，在脚本的其他任何地方都不能被改变。一个常量由英文字母、下划线、和数字组成,但数字不能作为首字母出现。常量在定义后，默认是全局变量，可以在整个运行的脚本的任何地方使用。 注：常量名不需要加 $ 修饰符。

PHP 魔术常量

PHP 向它运行的任何脚本提供了大量的预定义常量。不过很多常量都是由不同的扩展库定义的，只有在加载了这些扩展库时才会出现，或者动态加载后，或者在编译时已经包括进去。

魔术常量它们的值随着它们在代码中的位置改变而改变。

PHP 命名空间(namespace)

PHP 命名空间(namespace) 是在PHP 5.3中加入。 它可以解决这两类问题：一是用户编写的代码与PHP内部的类/函数/常量或第三方类/函数/常量之间的名字冲突。 二是为很长的标识符名称(通常是为了缓解第一类问题而定义的)创建一个别名（或简短）的名称，提高源代码的可读性。

PHP 文件处理

模式 描述
r 只读。在文件的开头开始。
r+ 读/写。在文件的开头开始。
w 只写。打开并清空文件的内容；如果文件不存在，则创建新文件。
w+ 读/写。打开并清空文件的内容；如果文件不存在，则创建新文件。
a 追加。打开并向文件末尾进行写操作，如果文件不存在，则创建新文件。
a+ 读/追加。通过向文件末尾写内容，来保持文件内容。
x 只写。创建新文件。如果文件已存在，则返回 FALSE 和一个错误。
x+ 读/写。创建新文件。如果文件已存在，则返回 FALSE 和一个错误。

PHP 错误处理

错误级别值 常量 描述
2 E_WARNING 非致命的 run-time 错误。不暂停脚本执行。
8 E_NOTICE run-time 通知。在脚本发现可能有错误时发生，但也可能在脚本正常运行时发生。
256 E_USER_ERROR 致命的用户生成的错误。错误无法恢复。脚本执行被中断。这类似于程序员使用 PHP 函数 trigger_error() 设置的 E_ERROR。
512 E_USER_WARNING 非致命的用户生成的警告。脚本执行不被中断。这类似于程序员使用 PHP 函数 trigger_error() 设置的 E_WARNING。
1024 E_USER_NOTICE 用户生成的通知(默认)。在脚本发现可能有错误时发生，但也可能在脚本正常运行时发生。这类似于程序员使用 PHP 函数 trigger_error() 设置的 E_NOTICE。
4096 E_RECOVERABLE_ERROR 可捕获的致命错误。类似 E_ERROR，但可被用户定义的处理程序捕获。（参见 set_error_handler()）
8191 E_ALL 所有错误和警告。（在 PHP 5.4 中，E_STRICT 成为 E_ALL 的一部分）
